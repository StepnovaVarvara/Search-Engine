Если дергаем контроллер запуска полной индексации (/startIndexing), то:
    1. в случае успеха
        1.1) создает в таблице site в БД запись со статусом INDEXING (+++)
        1.2) обновляет время при добавлении каждой page (+++)
        1.3) по завершении обхода статус - INDEXED (+++)
        1.4) если возникла ошибка, то статус - FAILED + сохраняем текст ошибки (как проверить?)
        1.5) заполняет таблицы lemma и index (+++)
        1.6) формат ответа - result: true (+++)

    2. в случае ошибки
        2.1) полная переиндексация - начать процесс индексации заново (+++)
        2.2) формат ответа - result: false, error: "Индексация уже запущена" (+++)

    Результат - в БД все три сайта со статусом INDEXED (+++)

Если дергаем контроллер остановки текущей индексации (/stopIndexing), то:
    1. в случае успеха
        1.1) останавливает поток (+++)
        1.2) сайты, которые не обошли, добавляем в БД со стататусом FAILED + "Индексация остановлена пользователем" (+++)
        1.3) формат ответа - result: true (+++)

    2. в случае ошибки
        2.1) формат ответа - result: false, error: "Индексация не запущена" (+++)

    Результат - в БД необойденные сайты со статусом FAILED (+++)

Если дергаем контроллер запуска индексации страницы (/indexPage), то:
    1. в случае успеха
        1.1) создает в таблице site в БД / привязка page к site (+++)
        1.2) если страница уже была проиндексирована, то удаляет данные из БД в таблицах page, lemma и index (+++)
        1.3) заполняет таблицы lemma и index (+++)
        1.4) формат ответа - result: true (+++)

    2. в случае ошибки
        2.1) формат ответа - result: false,
             error: "Данная страница находится за пределами сайтов, указанных в конфигурационном файле" (+++)

Если дергаем контроллер запуска индексации страницы (/statistics), то:
    1) отображает сумму по сайтам, страницам и леммам (+++)
    2) отображает сайты из конфигурации с их статусами из БД (+++)
    3) у каждого сайта отображает корректные данные из БД (+++)
    4) статус сайта (INDEXED) (+++)

Если дергаем контроллер запуска индексации страницы (/search), то:
    1. в случае успеха
        1.1) получаем массив с объектами Data (+++)
        1.2) массив содержит инф-ию о: site, siteName, uri, title, snippet, relevance (+++)
        1.3) в snippet фрагмент текста выделен жирным (+++)
    2. в случае ошибки
        2.1) формат ответа - result: false, error: "Задан пустой поисковый запрос" (+++)

Если в процессе работы возникают ошибки, то:
    1. в случае успеха
        1.1) код ошибки 400 - Неправильный запрос (+++)
        1.2) код ошибки 500 - Внутренняя ошибка сервера (+++)
